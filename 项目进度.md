## 五、风险与应对
| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 小红书采集方案不稳定（平台反爬升级） | 高 | 采集功能部分不可用 | 备选方案：浏览器扩展手动导出 + 批量上传；或暂不上线小红书，标注"二期支持" |
| FFmpeg 渲染耗时超出预估 | 中 | 产出速度不达预期 | 优先实现 720p 快速模式；GPU 加速方案提前验证；**变速重编码会增加 1-3s，已在预估中留余量** |
| 阿里云 TTS 并发限制 | 低 | 配音生成排队 | 本地 TTS 引擎备选（edge-tts 开源方案）；配音设为可选，关闭时跳过 |
| 字幕样式预览与最终产出不一致 | 中 | 设计师信任度下降 | 用完全相同的样式参数（字体/颜色/位置）分别驱动 CSS 和 ASS，确保一一对应；上线前逐项对比验证 |
| 大文件上传超时 | 低 | 素材上传失败 | 前端分片上传 + 后端 MinIO multipart；断点续传 |
| **素材分析耗时超预期** | **中** | **新入库素材分析积压** | **已缩放到 160x90 分辨率计算，O(n) 复杂度可控；独立队列不阻塞主流程；监控 analysis_status='pending' 积压量，超阈值告警** |
| **素材类型推断准确率不足** | **中** | **混剪策略误判（如把说话类误判为动作类导致变速变声）** | **当前为启发式规则，准确率不需要 100%（_people_talking_ 有保守判定逻辑）；二期升级为 VLM 视频大模型；降级策略兜底** |
| **运动检测对静态素材误报高光** | **低** | **低运动素材被错误截取** | **自适应阈值已考虑（overall_score * 0.4）；无高光段时回退为整段使用；混剪决策引擎会综合判断** |
| **分析队列内存溢出** | **低** | **Worker 崩溃** | **单次只加载一个视频到内存，处理完立即释放；限制最大处理时长 10 分钟超时终止；OpenCV 使用 headless 版本** |
