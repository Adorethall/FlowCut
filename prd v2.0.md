## 一、产品概述 
### 1.1 背景与痛点 
当前广告设计团队在制作信息流广告视频时存在以下问题： 
- **素材获取低效**：设计师手动在抖音、快手等平台搜索下载素材，耗时且难以管理 
- **素材孤岛**：已下载素材散落在各人电脑本地，无法共享复用，重复下载率高 
- **素材利用盲目**：下载的素材无法快速定位高光时刻，混剪时全凭感觉截取，导致产出节奏拖沓或错过精彩画面 
- **剪辑重复劳动**：每条广告视频的剪辑流程高度相似（混剪→压字幕→配音配乐），但每次都需要手动操作，且缺乏针对不同类型素材（动作类、说话类）的差异化处理 
- **产出规模受限**：信息流广告需要大量素材做 A/B 测试，人工剪辑产能不足 
- **成片无预览**：批量产出后无法快速预览效果，只能逐个下载到本地播放，审核效率低 
### 1.2 产品定位 
FlowCut 是面向广告设计团队的**智能素材采集管理 + 自动化混剪流水线 + 成片预览审核**工具。核心目标是将「找素材→选素材→剪视频→看效果」的链路从人工串联变为系统驱动，通过**运动检测提取高光**与**智能混剪策略（变速/截取）**，实现广告视频的高质量、批量自动化产出与快速审核。 
### 1.3 核心价值 
| 维度 | 现状 | 目标 | 
|------|------|------| 
| 素材获取 | 手动搜索下载，30分钟/批 | 脚本自动采集+自动高光标注，5分钟/批 | 
| 素材管理 | 本地文件夹，无法共享 | 线上素材库，带类型标签与高光可视化，多人协作 | 
| 视频剪辑 | 单条手动剪辑，20分钟/条 | 智能策略工作流（自动变速/截取高光），批量产出 | 
| 成片审核 | 逐个下载本地播放，5分钟/条 | 在线预览播放器+剪辑方案追溯，30秒/条 | 
| 产出规模 | 日均5-10条 | 日均50-100条 |

## 四、功能详细说明 
### 4.1 素材采集模块 
#### 4.1.1 采集脚本配置 
| 输入项 | 类型 | 说明 | 
|--------|------|------| 
| 搜索关键词 | 文本框 | 多个关键词用逗号分隔，如：`奶茶,珍珠奶茶,手打柠檬茶` | 
| 目标平台 | 多选按钮 | 抖音 / 快手 / B站 / 小红书，至少选 1 个 | 
| 每关键词下载数量 | 下拉选择 | 5 / 10 / 20 / 30 / 50 | 
| 视频时长范围 | 下拉选择（锁定） | 3-60 秒 | 
| 最低分辨率 | 下拉选择（锁定） | 720p | 
#### 4.1.2 自动入库与分析规则 ★ 升级 
- 采集完成后自动调用 ffprobe 提取视频元信息（时长、分辨率、帧率、文件大小、编码格式） 
- 根据采集关键词自动打标签 
- 自动生成封面图（取第 1 秒帧） 
- 文件存储路径：`materials/{平台}/{年}/{月}/{日}/{md5}.mp4` 
- 封面存储路径：`covers/materials/{md5}.jpg` 
- **异步高光标注**：入库后自动投入分析队列，使用 OpenCV 进行运动检测，生成素材类型标签（食物制作/人物互动等）、运动评分及高光时段数据。**此过程不阻塞采集和后续使用。** 
### 4.2 素材库管理模块 
#### 4.2.1 搜索与筛选 
- 支持按标题、关键词、标签模糊搜索（防抖 300ms） 
- **筛选条件新增**：素材类型（多选标签按钮组：食物制作 / 人物互动 / 风景静态 / 产品展示 / 氛围场景 / 其他）。 
- 素材分析状态（可选筛选：已分析 / 待分析）。 
#### 4.2.2 素材卡片信息展示 ★ 升级 
卡片采用 9:12 竖版缩略图布局，展示信息： 
| 元素 | 位置 | 说明 | 
|------|------|------| 
| 视频封面 | 主体 | 9:12 竖版缩略图 | 
| 时长标签 | 右上角 | 黑底白字，如 `0:08` | 
| 来源标识 | 左上角 | 平台图标 + 名称 | 
| **素材类型标签** | **左下角** | **如「动作类」「人物类」，带对应颜色底色，待分析时显示「待分析」** | 
| **高光指示条** | **封面底部** | **一条细长进度条，用主色高亮标记检测到的高光时段区间** | 
| 悬浮播放按钮 | 居中 | hover 时显示半透明圆形播放按钮 | 
| 勾选按钮 | 右下角 | 点击切换选中/取消 | 
| 标题 | 封面下方 | 单行截断 | 
| 元信息行 | 标题下方 | 分辨率 · 文件大小 · 运动评分(如已分析) | 
#### 4.2.3 素材预览模态框 ★ 升级 
点击卡片打开预览模态框： 
| 区域 | 内容 | 
|------|------| 
| 视频区 | 16:9 比例展示封面图 + 居中播放按钮 | 
| 进度条 | 底部细进度条（叠加高光时段高亮色块） | 
| 元信息 | 四宫格卡片：时长 / 分辨率 / 大小 / **素材类型** | 
| 高光可视化 | 封面下方展示完整的高光时段条（鼠标悬浮显示具体时间与评分） | 
| 操作按钮 | 「下载素材」「加入工作流」「重新分析」（分析失败时显示） | 
### 4.3 自动化剪辑工作流模块 
#### 4.3.1 Step 1 — 主题选择 
- 8 个预设主题卡片，显示主题图标、名称、素材数量、关联关键词。 
- **素材数量区分**：显示「总素材 50 · 已分析 45」，帮助用户判断素材成熟度。 
#### 4.3.2 Step 2 — 素材筛选与编排 
- 左侧可选池素材列表项增加「素材类型」标签列。 
- 悬浮缩略图时，缩略图底部浮现高光指示条。 
#### 4.3.3 Step 3 — 参数配置 ★ 新增混剪策略 
分为五个配置组（前四个同原版，每组有独立开关）： 
**混剪策略设置（核心新增）** 
| 参数 | 控件 | 选项 | 默认值 | 说明 | 
|------|------|------|--------|------| 
| 混剪风格 | 单选卡片组 | 快节奏 / 常规 / 慢节奏 | 常规 | 决定单段素材目标时长（快2-4s，常3-6s，慢5-10s） | 
| 允许变速 | 开关 | — | 开启 | 开启后，非人物说话类素材允许加速处理 | 
| 最大变速倍率 | 按钮组 | 2x / 3x / 5x | 3x | 配合「允许变速」使用，5x适合快闪类视频 | 
| 优先使用高光 | 开关 | — | 开启 | 优先从检测到的高光时段截取素材 | 
> **系统智能处理说明（对用户隐藏细节）**：当用户开启变速时，系统会自动识别「人物说话类」素材并强制原速处理，避免变声；对于未完成高光分析的素材，系统会自动降级为「中间截取」策略，不阻塞任务。 
**其他配置组** 
（配音、字幕、配乐、转场与输出设置保持与 v2.0 PRD 一致，略） 
#### 4.3.4 Step 4 — 确认提交 
三栏摘要卡片新增一栏： 
| 卡片 | 内容 | 
|------|------| 
| 主题与素材 | 主题图标+名称、已选素材数量、总时长 | 
| **混剪策略** | **风格、变速开关及倍率、高光优先开关** | 
| 配音/字幕/配乐 | 各项启用状态及关键参数 | 
| 输出配置 | 数量、时长、比例、分辨率、格式 | 
### 4.4 任务管理模块 
#### 4.4.1 任务卡片信息 ★ 升级 
| 元素 | 说明 | 
|------|------| 
| 图标 | 视频图标，灰色底 | 
| 任务名称 | 如「奶茶_混剪_1430」 | 
| 状态标签 | 色底 + 脉冲圆点 + 文字 | 
| 配置摘要 | 5条 × 15s · 9:16 · 1080p · **快节奏** · 素材5条 · 时间 | 
| 进度条（处理中） | 渐变色填充条 + 百分比 + 阶段文字 | 
#### 4.4.2 实时进度阶段文字更新 
| 进度区间 | 文字 | 
|----------|------| 
| 0-10% | 正在分析素材与规划剪辑方案... | 
| 10-30% | 正在生成配音... | 
| 30-60% | 正在智能混剪（变速/截取高光）视频片段... | 
| 60-75% | 正在合成转场效果... | 
| 75-85% | 正在压制字幕... | 
| 85-95% | 正在混入配乐... | 
| 95-99% | 正在编码输出... | 
| 100% | 全部完成 | 
### 4.5 成片预览模块 
#### 4.5.1 产出列表侧栏 ★ 升级 
- 列表项缩略卡片底部新增**剪辑方案简览图标**（如剪刀带闪电图标表示使用了变速）。 
- 点击视频播放时，右侧列表下方或悬浮面板可展开**「剪辑方案追溯」**： 
  - 以时间轴形式展示该条视频由哪些素材拼接而成。 
  - 每段标注处理策略：`[素材A] 5x变速`、`[素材B] 截取高光 2.1s-7.5s`、`[素材C] 原速截取`。 
  - 目的：帮助设计师快速了解产出结构，便于针对性调整后续任务配置。

## 五、非功能需求 
### 5.1 性能指标 
| 指标 | 目标 | 
|------|------| 
| 并发浏览 | 支持 10 人同时在线浏览素材库 | 
| 素材存储 | 初期支持 2TB，可水平扩展 | 
| 视频封面加载 | 首帧加载 < 2s | 
| 搜索响应 | < 500ms | 
| **素材分析耗时** | **10s视频 < 2s；30s视频 < 4s；60s视频 < 6s（异步不阻塞）** | 
| 单条视频渲染（1080p, 15s） | < 25s（含变速重编码开销） | 
| 单条视频渲染（720p, 15s） | < 15s | 
| 成片预览打开 | < 1s（封面图已缓存时） | 
| 任务进度推送延迟 | < 500ms（SSE） | 
| 日产出吞吐（单 Worker） | ≥ 2800 条 | 
### 5.2 数据安全 
| 风险 | 措施 | 
|------|------| 
| 素材文件外部直链访问 | MinIO 全部私有 Bucket，预签名 URL 访问，有效期限制 | 
| 接口未授权 | JWT Token 认证 | 
| 文件上传恶意内容 | 后缀白名单 + ffprobe 合法性校验 + OpenCV 文件头校验 | 
| FFmpeg 命令注入 | subprocess 列表参数传递，禁止 shell 字符串拼接 | 
| 大文件上传 OOM | 流式上传到 MinIO，不经后端内存 | 
| 采集被封 | 单平台并发 ≤ 2，请求间隔随机 2-5s | 
| **恶意视频触发分析 OOM** | **限制最大处理时长（10分钟），超时强制终止进程** | 
### 5.3 可用性与容错 
| 指标 | 目标 | 
|------|------| 
| 服务可用性 | 99.5%（月度） | 
| 单条视频渲染失败 | 不影响同任务其他条，记录失败原因 | 
| **未分析素材参与混剪** | **自动降级为普通截取策略，不报错阻塞任务，任务详情中记录降级日志** |

## 八、数据结构概要 
### 8.1 核心实体（重点变更字段） 
| 实体 | 说明 | 关键新增/变更字段 | 
|------|------|----------| 
| Material | 素材 | **`type_tag`** (枚举: food_action/people_talking等)、**`highlight_segments`** (JSONB高光时段列表)、**`motion_score`** (运动评分)、**`analysis_status`** (分析状态) | 
| Task | 剪辑任务 | `config` 中 **`random_strategy` 废弃，替换为 `mix_strategy`** (JSONB) | 
| TaskOutput | 产出视频 | **`edit_plan`** (JSONB，记录本条视频实际使用的剪辑方案，用于前端追溯展示) | 
### 8.2 配置 JSON 结构变更 
**task.config 新增 `mix_strategy`** 
```json 
{ 
  // ... 其他配置保留 ... 
  "mix_strategy": { 
    "style": "normal",       // fast / normal / slow 
    "allow_speedup": true,   // 是否允许变速 
    "max_speed": "3x",       // 2x / 3x / 5x 
    "prefer_highlights": true // 优先使用高光片段 
  } 
} 
``` 
**material.highlight_segments 结构** 
```json 
[ 
  { "start": 3.2, "end": 8.5, "score": 45.3 }, 
  { "start": 11.0, "end": 14.2, "score": 32.1 } 
] 
``` 
**task_output.edit_plan 结构（用于前端追溯）** 
```json 
[ 
  { "material_id": 1, "strategy": "speedup", "source_start": 0.0, "source_end": 15.0, "output_duration": 3.0, "speed": 5.0 }, 
  { "material_id": 6, "strategy": "cut_highlight", "source_start": 2.1, "source_end": 7.5, "output_duration": 5.4, "speed": 1.0 } 
] 
```
