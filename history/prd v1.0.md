## 一、产品概述
### 1.1 背景与痛点
当前广告设计团队在制作信息流广告视频时存在以下问题：
- **素材获取低效**：设计师手动在抖音、快手等平台搜索下载素材，耗时且难以管理
- **素材孤岛**：已下载素材散落在各人电脑本地，无法共享复用，重复下载率高
- **剪辑重复劳动**：每条广告视频的剪辑流程高度相似（混剪→压字幕→配音配乐），但每次都需要手动操作
- **产出规模受限**：信息流广告需要大量素材做 A/B 测试，人工剪辑产能不足
- **成片无预览**：批量产出后无法快速预览效果，只能逐个下载到本地播放，审核效率低
### 1.2 产品定位
FlowCut 是面向广告设计团队的**素材采集管理 + 自动化混剪流水线 + 成片预览审核**工具，核心目标是将「找素材→选素材→剪视频→看效果」的链路从人工串联变为系统驱动，实现广告视频的批量自动化产出与快速审核。
### 1.3 核心价值
| 维度 | 现状 | 目标 |
|------|------|------|
| 素材获取 | 手动搜索下载，30分钟/批 | 脚本自动采集，5分钟/批 |
| 素材管理 | 本地文件夹，无法共享 | 线上素材库，多人协作 |
| 视频剪辑 | 单条手动剪辑，20分钟/条 | 自动化工作流，批量产出 |
| 成片审核 | 逐个下载本地播放，5分钟/条 | 在线预览播放器，30秒/条 |
| 产出规模 | 日均5-10条 | 日均50-100条 |

## 四、功能详细说明
### 4.1 素材采集模块
#### 4.1.1 采集脚本配置
| 输入项 | 类型 | 说明 |
|--------|------|------|
| 搜索关键词 | 文本框 | 多个关键词用逗号分隔，如：`奶茶,珍珠奶茶,手打柠檬茶` |
| 目标平台 | 多选按钮 | 抖音 / 快手 / B站 / 小红书，至少选 1 个 |
| 每关键词下载数量 | 下拉选择 | 5 / 10 / 20 / 30 / 50 |
| 视频时长范围 | 下拉选择（锁定） | 3-60 秒 |
| 最低分辨率 | 下拉选择（锁定） | 720p |
**去重策略**：计算视频文件 MD5（前 10MB + 尾部 1MB 快速计算），与素材库中已有记录比对，重复则跳过。
**执行方式**：后端 Celery 异步执行，前端通过进度条 + 文字实时展示采集状态。
#### 4.1.2 自动入库规则
- 采集完成后自动调用 ffprobe 提取视频元信息（时长、分辨率、帧率、文件大小、编码格式）
- 根据采集关键词自动打标签
- 自动生成封面图（取第 1 秒帧）
- 文件存储路径：`materials/{平台}/{年}/{月}/{日}/{md5}.mp4`
- 封面存储路径：`covers/materials/{md5}.jpg`
#### 4.1.3 采集进度展示
| 展示项 | 说明 |
|--------|------|
| 总进度条 | 百分比进度，渐变色填充 |
| 当前阶段 | 如「正在采集：奶茶 - 抖音 (第 3 / 10 条)」 |
| 采集结果统计 | 发现总数 / 实际下载量 / 去重跳过量 |

### 4.2 素材库管理模块
#### 4.2.1 搜索
- 支持按标题、关键词、标签模糊搜索
- 输入防抖 300ms，避免频繁请求
- 搜索框带搜索图标，空状态引导
#### 4.2.2 筛选条件
| 筛选维度 | 类型 | 交互形式 | 选项 |
|----------|------|----------|------|
| 来源平台 | 多选 | 标签按钮组 | 抖音 / 快手 / B站 / 小红书 |
| 标签 | 单选 | 标签按钮组 + "全部" | 动态（从已有标签中生成） |
> 注：PRD v1.0 中的时长范围滑块和分辨率多选移至二期，当前版本保持简洁。
#### 4.2.3 素材卡片信息展示
卡片采用 9:12 竖版缩略图布局（适配信息流视频主流比例），展示信息：
| 元素 | 位置 | 说明 |
|------|------|------|
| 视频封面 | 主体 | 9:12 竖版缩略图 |
| 时长标签 | 右上角 | 黑底白字，如 `0:08` |
| 来源标识 | 左上角 | 平台图标 + 名称，使用平台品牌色 |
| 悬浮播放按钮 | 居中 | hover 时显示半透明圆形播放按钮 |
| 勾选按钮 | 左下角 | 点击切换选中/取消，选中态为主色填充 |
| 标题 | 封面下方 | 单行截断 |
| 元信息行 | 标题下方 | 分辨率 · 文件大小 · 上传时间，灰色小字 |
| 标签 | 底部 | 小号标签药片 |
#### 4.2.4 素材预览
点击卡片播放区域打开预览模态框：
| 区域 | 内容 |
|------|------|
| 视频区 | 16:9 比例展示封面图 + 居中播放按钮（模拟） |
| 进度条 | 底部细进度条 |
| 元信息 | 四宫格卡片：时长 / 分辨率 / 大小 / 来源 |
| 标签 | 标签药片列表 |
| 操作按钮 | 「下载素材」「加入工作流」 |
#### 4.2.5 批量操作
- 勾选素材后顶部出现操作栏：「加入工作流 (N)」+「取消选择」
- 点击「加入工作流」自动跳转到工作流 Step 2，已选素材预填入右侧已选池
- 自动推断主题：按素材关键词频次取最高者作为推荐主题

### 4.3 自动化剪辑工作流模块
#### 4.3.1 顶部步骤条
四个步骤横向排列，用圆点 + 连线表示：
- **未到达**：灰色边框圆点 + 灰色文字
- **当前**：主色填充圆点（带发光阴影）+ 主色文字
- **已完成**：主色填充圆点（内嵌对勾）+ 灰色文字
#### 4.3.2 Step 1 — 主题选择
- **预设主题**：8 个主题卡片（奶茶 / 火锅 / 烧烤 / 护肤 / 健身 / 旅游 / 穿搭 / 数码），每个显示主题图标、名称、素材数量、关联关键词
- **自定义主题**：右上角输入框 + 添加按钮，输入主题名称即可新增（自动生成默认文案）
- **选中态**：主色边框 + 主色浅底 + 外发光
- **校验**：必须选择主题才可进入下一步
#### 4.3.3 Step 2 — 素材筛选与编排
左右分栏布局（3:2 比例）：
**左侧 — 可选素材池**
- 标题显示「可选素材」+ 当前主题名
- 根据主题关键词自动匹配素材库中的素材
- 支持在当前结果基础上进一步搜索筛选
- 每条素材显示缩略图、标题、时长、分辨率、大小、标签
- 点击整行添加到右侧（已添加的显示主色边框 + 对勾图标）
**右侧 — 已选素材池**
- 标题显示「已选素材」+ 计数 + 最低要求提示
- 每条素材显示缩略图、标题、时长
- 左侧上下箭头按钮调整排序
- 右侧 × 按钮移除
- 空状态：虚线边框 + 提示文案
- **校验**：至少选择 3 条素材才可进入下一步
#### 4.3.4 Step 3 — 参数配置
分为四个配置组，每组左侧有竖线装饰（hover 变主色），每组有独立的启用/禁用开关：
**配音设置**
| 参数 | 控件 | 选项 | 默认值 |
|------|------|------|--------|
| 启用 | 开关 | — | 开启 |
| 配音风格 | 下拉选择 | 活力女声 / 沉稳男声 / 可爱女声 / 磁性男声 | 活力女声 |
| 语速 | 下拉选择 | 0.8x / 1.0x / 1.2x / 1.5x | 1.0x |
| 音量 | 滑块 | 0-100，右侧实时显示数值 | 80 |
**字幕设置**
| 参数 | 控件 | 选项 | 默认值 |
|------|------|------|--------|
| 启用 | 开关 | — | 开启 |
| 位置 | 下拉选择 | 底部居中 / 底部左对齐 / 顶部居中 | 底部居中 |
| 字体 | 下拉选择 | 思源黑体 / 思源宋体 / 站酷快乐体 | 思源黑体 |
| 字号 | 下拉选择 | 小 / 中 / 大 | 中 |
| 颜色 | 下拉选择 | 白色 / 黄色 / 青色 | 白色 |
| 描边 | 开关 | — | 开启 |
> 注：字幕配置会保存到任务记录中，成片预览时据此还原字幕样式。
**配乐设置**
| 参数 | 控件 | 选项 | 默认值 |
|------|------|------|--------|
| 启用 | 开关 | — | 开启 |
| 音乐风格 | 下拉选择 | 轻快 / 抒情 / 动感 / 古风 / 电子 | 轻快 |
| 音量 | 滑块 | 0-100 | 30 |
**转场与输出**
| 参数 | 控件 | 选项 | 默认值 |
|------|------|------|--------|
| 转场效果 | 下拉选择 | 溶解 / 滑动 / 缩放 / 模糊 / 无 | 溶解 |
| 转场时长 | 下拉选择 | 0.3s / 0.5s / 0.8s / 1.0s | 0.5s |
| 输出数量 | 下拉选择 | 3 / 5 / 10 / 15 / 20 | 5 |
| 单条时长 | 下拉选择 | 10s / 15s / 20s / 30s | 15s |
| 画面比例 | 下拉选择 | 9:16 / 16:9 / 1:1 | 9:16 |
| 分辨率 | 下拉选择 | 720p / 1080p | 1080p |
#### 4.3.5 Step 4 — 确认提交
三栏摘要卡片 + 素材预览条：
| 卡片 | 内容 |
|------|------|
| 主题与素材 | 主题图标+名称、已选素材数量、总时长 |
| 配音/字幕/配乐 | 各项启用状态及关键参数 |
| 输出配置 | 数量、时长、比例、分辨率、格式 |
底部素材预览条：横向滚动展示所有已选素材缩略图。
操作：「上一步」+「提交生成」（主色按钮，带火箭图标）。

### 4.4 任务管理模块
#### 4.4.1 状态标签页
顶部五个筛选标签：全部 / 排队中 / 处理中 / 已完成 / 失败，每个带数量角标。
#### 4.4.2 任务状态流转
```
排队中 ──→ 处理中 ──→ 已完成
                │
                └──→ 失败 ──→ 重试 ──→ 排队中
```
#### 4.4.3 任务卡片信息
| 元素 | 说明 |
|------|------|
| 图标 | 视频图标，灰色底 |
| 任务名称 | 如「奶茶_混剪_1430」 |
| 状态标签 | 色底 + 脉冲圆点 + 文字 |
| 配置摘要 | 5条 × 15s · 9:16 · 1080p · 素材5条 · 时间 |
| 进度条（处理中） | 渐变色填充条 + 百分比 + 阶段文字 |
| 操作按钮 | 已完成：「预览成片」+「下载全部」；失败：「重试」；通用：「删除」 |
#### 4.4.4 实时进度推送
处理中的任务通过 SSE（Server-Sent Events）实时推送进度，前端监听后更新进度条和阶段文字，无需手动刷新。
阶段文字示例：
| 进度区间 | 文字 |
|----------|------|
| 0-10% | 正在分析素材... |
| 10-30% | 正在生成配音... |
| 30-55% | 正在混剪视频片段... |
| 55-75% | 正在压制字幕... |
| 75-90% | 正在混入配乐... |
| 90-99% | 正在编码输出... |
| 100% | 全部完成 |
#### 4.4.5 产出操作
- **预览成片**：打开成片预览模态框（详见 4.5）
- **下载全部**：后端异步打包为 ZIP，返回下载链接，Toast 提示「正在打包 N 条视频下载...」
- **单条下载**：在成片预览中提供（详见 4.5）

#### 4.5.3 视频播放器
**视频画面区**
- 宽高比根据任务配置的输出比例自适应（9:16 / 16:9 / 1:1）
- 封面由该条视频使用的多个素材缩略图拼接而成（模拟混剪效果）
- 封面上叠加半透明黑色遮罩层
**大播放按钮**
- 居中圆形按钮，主色背景 + 白色播放图标
- hover 放大 + 主色外发光
- 播放中自动隐藏（淡出缩小），暂停时重新显示
- 点击切换播放/暂停
**底部控制条**
- 默认隐藏，鼠标 hover 播放区时从底部渐显
- **进度条**：细条状，主色渐变填充；hover 时高度增加；hover 时显示拖拽圆点（主色发光）；点击任意位置跳转
- **左侧**：播放/暂停小按钮（主色 hover） + 当前时间/总时长（等宽字体）
- **右侧**：上一条 / 下一条切换按钮 + 当前文件名（灰色小字）
**播放逻辑**
- 播放时进度条每 100ms 更新一次
- 播放到末尾自动暂停
- 暂停后点击大播放按钮或控制条播放按钮继续播放
- 切换视频时自动暂停当前、重置进度、更新封面
**自动连播**
- 当前视频播放到末尾后，暂停 0.7 秒自动切换到下一条并开始播放
- 最后一条播完后停止，不循环
#### 4.5.4 字幕叠加层
**显示逻辑**
- 仅在任务配置中「字幕启用」时显示
- 根据字幕时间线数据（`subtitle_segments`），计算当前播放时间应显示哪段文案
- 匹配到文案时淡入显示（opacity 0→1，150ms 过渡）
- 无匹配时淡出隐藏（opacity 1→0，150ms 过渡）
**样式还原**
字幕样式完全根据任务提交时的字幕配置还原：
| 配置项 | 还原方式 |
|--------|----------|
| 位置 | CSS `position: absolute` + `bottom` / `top` / `text-align` |
| 字体 | CSS `font-family` 映射（思源黑体 → Source Han Sans SC 等） |
| 字号 | CSS `font-size` 映射（小 15px / 中 17px / 大 20px） |
| 颜色 | CSS `color` + `text-shadow` 实现发光效果 |
| 描边 | CSS `-webkit-text-stroke` |
**文案来源**
每条产出视频关联一条广告文案（如「这杯奶茶绝了」），文案被均分为 3-5 段，每段分配起止时间，形成字幕时间线。文案从预设文案库中按顺序取用，产出数量超过文案数量时循环取用。
#### 4.5.5 产出列表侧栏
**列表项（缩略卡片）**
- 9:14 比例缩略图（素材拼接封面）
- 左上角主色底白字编号标签（如 `#1`）
- 右下角黑底白字时长标签
- hover 时显示半透明播放图标覆盖层
- 下方显示广告文案（单行截断）+ 文件大小
**交互**
- 点击切换到对应视频播放
- 当前播放项：主色边框 + 主色浅底 + 外发光（`active-thumb` 样式）
- 切换时自动滚动到可视区域（`scrollIntoView({ block: 'nearest' })`）
#### 4.5.6 顶栏
| 元素 | 说明 |
|------|------|
| 左侧 | 主题图标（主题色）+ 任务名 + 产出统计（N 条 · 时长 · 比例 · 分辨率） |
| 右侧 | 「打包下载」按钮 + 关闭按钮 |

## 五、非功能需求
### 5.1 性能指标
| 指标 | 目标 |
|------|------|
| 并发浏览 | 支持 10 人同时在线浏览素材库 |
| 素材存储 | 初期支持 2TB，可水平扩展 |
| 视频封面加载 | 首帧加载 < 2s |
| 搜索响应 | < 500ms |
| 单条视频渲染（1080p, 15s） | < 25s |
| 单条视频渲染（720p, 15s） | < 15s |
| 成片预览打开 | < 1s（封面图已缓存时） |
| 任务进度推送延迟 | < 500ms（SSE） |
| 批量任务排队 | 支持 5 个任务同时排队 |
| 日产出吞吐（单 Worker） | ≥ 3000 条 |
### 5.2 数据安全
| 风险 | 措施 |
|------|------|
| 素材文件外部直链访问 | MinIO 全部私有 Bucket，预签名 URL 访问，有效期限制 |
| 接口未授权 | JWT Token 认证 |
| 文件上传恶意内容 | 后缀白名单 + ffprobe 合法性校验 |
| FFmpeg 命令注入 | subprocess 列表参数传递，禁止 shell 字符串拼接 |
| 大文件上传 OOM | 流式上传到 MinIO，不经后端内存 |
| 采集被封 | 单平台并发 ≤ 2，请求间隔随机 2-5s |
### 5.3 可用性
| 指标 | 目标 |
|------|------|
| 服务可用性 | 99.5%（月度） |
| 单条视频渲染失败 | 不影响同任务其他条，记录失败原因 |
| 任务整体失败 | 支持一键重试 |

## 七、技术方案概述
| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Element Plus + Pinia | 管理后台型 SPA |
| 后端 | Python FastAPI + SQLAlchemy | 异步 API，ORM 操作 |
| 数据库 | PostgreSQL 15 | 结构化数据持久化 |
| 文件存储 | MinIO | 视频文件 + 封面图私有存储 |
| 缓存/队列 | Redis 7 | 缓存 + Celery Broker + Pub/Sub |
| 任务队列 | Celery 5 | 采集队列（并发2）+ 渲染队列（并发=CPU核数） |
| 视频处理 | FFmpeg 6 | 拼接 / 转场 / 字幕(ASS) / 音频混合 |
| 语音合成 | 阿里云智能语音 TTS | 四种风格配音 |
| 采集脚本 | yt-dlp + Playwright | 抖音/快手/B站直接下载，小红书浏览器提取 |
| 实时通信 | SSE (Server-Sent Events) | 任务进度推送 |
| 部署 | Docker Compose + Nginx | 一键编排部署 |

## 八、数据结构概要
### 8.1 核心实体
| 实体 | 说明 | 关键字段 |
|------|------|----------|
| Material | 素材 | id, title, file_key, md5_hash, duration, width, height, source, tags[] |
| MaterialTag | 素材-标签关联 | material_id, tag |
| Task | 剪辑任务 | id(UUID), name, theme, status, progress, config(JSONB), subtitle_config(JSONB) |
| TaskMaterial | 任务-素材关联 | task_id, material_id, sort_order |
| TaskOutput | 产出视频 | task_id, index, file_key, duration, copy_text, subtitle_segments(JSONB) |
| CollectTask | 采集任务 | id(UUID), keywords[], platforms[], status, progress |
| CollectResult | 采集明细 | collect_task_id, keyword, platform, source_url, status |
### 8.2 配置 JSON 结构
**task.config（剪辑完整配置）**
```json
{
  "voice": { "enabled": true, "style": "活力女声", "speed": "1.0", "volume": 80 },
  "subtitle": { "enabled": true, "position": "底部居中", "font": "思源黑体", "size": "中", "color": "白色", "stroke": true },
  "music": { "enabled": true, "style": "轻快", "volume": 30 },
  "transition": { "type": "溶解", "duration": "0.5s" },
  "output": { "count": 5, "duration": "15s", "ratio": "9:16", "resolution": "1080p" },
  "random_strategy": "random_segments"
}
```
**task.subtitle_config（预览用字幕配置副本）**
```json
{
  "enabled": true,
  "position": "底部居中",
  "font": "思源黑体",
  "color": "白色",
  "stroke": true
}
```
**task_output.subtitle_segments（字幕时间线）**
```json
[
  { "text": "这杯奶茶", "start": 0.0, "end": 3.0 },
  { "text": "绝了", "start": 3.0, "end": 6.0 }
]
```
